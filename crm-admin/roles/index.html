기획 반영 메모 · 미확정 항목

  • SNB 너비 — 280px로 설정. '메뉴 관리' 목업과 달리 SNB가 항상 노출되는 페이지 구조이므로 합리적 폭으로 적용. 디자인 확정 필요.
  • SNB 단일 목록 — 기획서 v11(2026-05-22) 2.4 명시. 관리자/사용자 레벨 섹션 구분 없음.
  • 수퍼 어드민 식별 — shield 아이콘 + 텍스트 inline — 기획서 v11 2.4 / 2.5 명시. purple-600 컬러로 적용. SNB·메인 헤더 모두 동일 패턴.
  • 메뉴 목록 데이터 소스 — 메뉴 목록은 [메뉴 관리]에 등록된 메뉴를 그대로 노출 (기획서 1.3 / 3.1.5 / 5절 명시). 본 페이지에서는 메뉴 추가·삭제·순서 변경 불가. 메뉴 설정 탭 표 하단에 안내 문구로 명시.
  • 메뉴 설정 탭 — 1 Depth 셀 rowspan 처리 — 1 Depth 그룹화를 위한 시각적 묶음. 기획서에 별도 명시 없으나 사용자 가독성 위해 합리적 적용.
  • 조회 권한 고정 ON 토글 디자인 — opacity 낮춤(locked) + cursor not-allowed + '고정 ON · 변경 불가' 캡션. 일반 disabled(회색)와 시각적으로 구분. 디자인 확정 필요.
  • 접근 OFF 메뉴 카드 비활성화 — 카드 전체 opacity 0.85 + bg gray-25 + 모든 토글 disabled + '메뉴 접근 불가' 배지. 기획서 3.2.4 명시.
  • 메뉴별 추가 권한 (3.2.6 확장) — 기본 4종과 분리 노출. 점선 구분 + '메뉴별 추가 권한' 라벨 + '확장' 배지. MVP 시점 노출 여부 확정 필요(시안에는 변형 C 두 번째 카드에 예시 포함).
  • 권한 설정 탭 — 메뉴 카드 정렬 순서 — 기획서에 명시 없음. 본 시안은 메뉴 관리 페이지의 1 Depth · 2 Depth 순서를 따름. 정렬 정책 확정 필요(접근 ON 우선/메뉴 순서/알파벳 등).
  • 접근 OFF 메뉴 카드의 표시 범위 — 본 시안은 카드 전체 노출. 운영자가 변경 필요 시 한눈에 파악 가능하나, 메뉴가 많아질 경우 스크롤 부담. 접힘(collapse) 옵션 또는 필터 검토 필요.
  • SNB에서 역할 전환 시 미저장 변경 처리 — 기획서 4.1 "토글 클릭 시 자동 저장"이므로 미저장 상태 자체가 존재하지 않음. 단, 자동 저장 도중 다른 역할 클릭 시 동작 정책 미정의 (debounce / 락 처리 등 BE 협의 필요).
  • 권한 설정 탭 진입 시 데이터 로딩 상태 — 메뉴 수가 많은 경우 로딩 스피너 또는 스켈레톤 노출 정책 미정의.
  • 검색·필터 — 기획서에 명시 없음. 메뉴 수가 많아질 경우 검색·필터 추가 검토 필요(메뉴명/1 Depth 필터 등).
  • 토스트 위치·노출 시간 — '메뉴 관리' 목업과 동일하게 미정의. 시안 확보 후 적용.
  • 권한 변경 이력 / 감사 로그 — 기획서 범위 외. 운영 안정성 차원에서 추후 확장 검토 필요.
  • 역할 설명 출처 — 기획서 v11 2.5 명시 — '역할 관리' 페이지에서 입력된 값을 그대로 표시. 본 페이지에서 수정 불가. 본 시안에서는 임시 텍스트 사용.
  • 'URL 직접 접근만' 조합 운영 위험 — 노출 OFF + 접근 ON 조합은 기획서 3.1.4에 허용되어 있으나, 운영 사고로 의도치 않게 설정될 가능성 있음. 운영 가이드 차원에서 검토 필요.

v1 작성 의도

  • 레이아웃 패턴 전환: '메뉴 관리'의 단일 테이블 구조와 달리, '역할별 권한 관리'는 SNB(역할 목록) + 메인(역할 상세 + 탭) 구조로 변경. 기획서 2.1 명시.
  • 모달 위주 → 상태 변형 위주: 토글 즉시 저장(4.1) 정책이므로 등록·수정 모달이 거의 없음. 시안 구조도 'v3 메뉴 관리'의 모달 8종 패턴이 아닌 상태별 변형 4종(A 미선택 / B 메뉴 설정 / C 권한 설정 / D 매트릭스)으로 구성.
  • 기획서 v11(2026-05-22) 정합성 반영: ① 역할 레벨 컨셉 폐기 → SNB 단일 목록 ② 수퍼 어드민 식별 → shield 아이콘(purple-600) + 텍스트 inline 표기 ③ 페이지 헤더 · 안내 박스 정의 신설 ④ 권한 설정 탭에 URL 표기 제거 (기획서 미정의).
  • 수퍼 어드민 선택 상태를 기본 화면으로 노출: 페이지 진입 시 기본 미선택 상태가 정상이나, 시안 검토 용이성을 위해 기본 영역에는 '수퍼 어드민' 선택 + 전체 ON 고정 상태를 노출. '역할 미선택' 시안은 [참고 시안 — 변형 A]에 별도 정의.
  • MVP 전체 메뉴 반영: 메뉴 설정 탭(기본 영역 + 변형 B)에 MVP 기준 23개 메뉴를 전체 노출. 메뉴 목록 데이터 소스는 [메뉴 관리]에 등록된 메뉴를 그대로 가져오며, 본 페이지에서 메뉴 추가·삭제 불가.
  • 조회 권한 고정 ON UI: 일반 disabled(회색)와 구분하기 위해 'on + locked' 상태로 시각화. 토글은 활성 색(blue-500)이되 opacity 낮춤 + '고정 ON · 변경 불가' 캡션 추가.
  • 접근 OFF 메뉴 카드 비활성화: 기획서 3.2.4 명시. 카드 전체 opacity 낮춤 + 배지 '메뉴 접근 불가' + 카드 내 모든 토글 disabled.
  • 조합 매트릭스 별도 표 정리: 노출/접근 조합(3.1.4), 접근/조회 조합(3.2.5)을 변형 D에 매트릭스 형태로 정리. 권한 동작 정책 검토 시 빠른 참조 가능.
